# kaaaaai.tools.hexo-bb-channel

Hexo plugin for rendering a Telegram channel as a dynamic microblog page.

The plugin only generates a static page shell. Posts are loaded from a separate API backend, so your blog does not need to rebuild when the Telegram channel updates.

## Install

```bash
npm install hexo-bb-channel
```

If the package is not published yet, install from GitHub:

```bash
npm install github:Kaaaaai/kaaaaai.tools.hexo-bb-channel
```

## Backend

## Hexo Config

Add this to your root `_config.yml`:

```yml
bb_channel:
  enable: true
  mode: client
  route: bb/
  title: 闲言碎语
  description: 这些片段可能来自于大脑皮层短暂兴奋后的捕捉 🤏
  api_base: https://your-tg-channel-api.vercel.app
  page_size: 20
```

Then run:

```bash
hexo clean
hexo generate
hexo server
```

Open `http://localhost:4000/bb/`.

## How It Works

```text
Hexo /bb/
  static HTML shell generated by this plugin

Browser
  fetches api_base/api/posts?page=1&page_size=20

tg-channel-api
  fetches Telegram public web pages
  parses and sanitizes posts
  caches payload in Upstash Redis
```

## Options

| Option | Default | Description |
| --- | --- | --- |
| `enable` | `true` | Enables the generated page. |
| `mode` | `client` | Dynamic client-rendered mode. |
| `route` | `bb/` | Output route. |
| `title` | `moments` | Page title rendered inside the content shell. |
| `description` | empty | Page description rendered under the title. |
| `api_base` | empty | Base URL of the deployed `tg-channel-api`. |
| `page_size` | `20` | Posts requested per API page. |

## Notes

- This plugin does not fetch Telegram during `hexo generate`.
- Use a public Telegram channel. Private channels require a different authorization model.
- If your theme already has a `/bb/` source page, remove it or change `route` to avoid duplicate output.
